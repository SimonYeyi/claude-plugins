# 评审循环流转机制 — 双向沟通循环（适用于所有评审场景）

```
评审Agent给出评审意见
    │
    ▼
主干Agent独立思考（不是盲目接受）
    │
    ├──认为有理 → 修改 → 反馈给评审Agent
    │               │
    │               ▼
    │         评审Agent重新核实
    │               │
    │               ├──同意（记录在案）──→ 评审通过
    │               │
    │               └──不同意（给出理由）──→ 打回主干Agent
    │
    └──认为无理 → 坚持己见 + 反馈不修改的具体理由
                    │
                    ▼
              评审Agent重新思考
                    │
                    ├──同意（记录在案）──→ 评审通过
                    │
                    └──不同意 ──→ 循环（最多5次）
                                │
                                └──5次后主控决断
```
