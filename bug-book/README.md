# bug-book

AI 错题集：让 AI 记住历史问题的现象、根因、解决方案，防止重复犯错。

## 功能

- **记录** — AI 感知问题后自动记录；小问题自动验证，复杂/二次问题后台直接记录，修复时统一审核
- **召回** — 改代码前自动查询相关 bug，主动预防
- **整理** — 归类、去重、清理失效条目、按重要性排序
- **搜索** — 关键词快速定位

## 安装

将 `bug-book` 目录添加到 Claude Code 的插件路径中。具体方式请参考 Claude Code 插件文档。

**注意**：`bug-book.db` 数据库文件不应该随项目提交，建议将其加入 `.gitignore`。每个开发者有独立的本地数据库。

## 使用

### 记录问题

当 AI 被纠正或发现问题时，触发 bug-record skill 记录问题信息。

- 预估分 < 30 的小问题（THRESHOLD_AUTO_VERIFY）：自动记录并验证
- 预估分 >= 30 的复杂问题：后台直接记录，修复时统一审核

**用户也可以主动记录**：直接说"帮我记录一个问题"（哪怕 AI 没有上下文），AI 会主动询问必要字段后写入。

### 查询问题

改代码前 AI 会自动查询相关 bug，或手动搜索：

```
有没有 XX 相关的 bug 记录？
```

### 整理错题集

定期整理错题集：

```
整理一下错题集
```

## 数据库

SQLite 数据库位于项目根目录 `bug-book.db`。每个项目有独立的数据库，互不干扰。
首次使用时自动初始化，无需手动操作。

## 评分体系

| 维度 | 权重 | 分值范围 | 说明 |
|------|------|----------|------|
| 功能重要性 | 2.0 | 0-10 | 功能越核心分越高 |
| 逻辑复杂度 | 1.5 | 0-10 | 涉及逻辑越复杂分越高 |
| 影响范围 | 1.0 | 0-10 | 影响多少文件/模块 |
| 修复难度 | 1.0 | 0-10 | 越难修复/耗时越长分越高 |
| 出现次数 | 1.0 | 累计 | 每次重复出现 +1 |
| 用户情绪 | 1.5 | 0-5 | 被纠正 +2，发火 +5 |
| 预防价值 | 2.0 | 0-10 | 值得被优先记住的程度 |
