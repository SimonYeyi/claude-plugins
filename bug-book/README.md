# bug-book

Claude Code 插件：AI 错题集系统，帮助 AI 记住项目中的历史问题，避免重复犯错。

## 功能特性

- **智能记录** — AI 发现问题或被纠正时自动记录，支持手动记录
- **主动召回** — 修改代码前自动检索相关历史问题，让 AI 避免重犯相同错误
- **智能整理** — 自动清理失效条目、归类相似问题、按重要性排序
- **快速搜索** — 通过关键词、标签、文件路径等多种方式检索历史问题

## 安装

1. 将 `bug-book` 目录复制到 Claude Code 的插件目录
2. 首次使用时会自动初始化数据库（位于项目根目录的 `bug-book.db`）

**注意**：数据库文件应加入 `.gitignore`，每个开发者使用独立的本地数据库。

## 基本使用

### 手动记录

用户可以主动要求记录问题：
```
帮我记录一个问题：[描述问题现象]
```

AI 会引导补充必要信息后自动记录。

### 查询历史问题

在修改代码前，可以询问相关历史问题：
```
有没有关于 session 管理的 bug 记录？
查看 auth 模块的历史问题
```

### 整理错题集

定期清理和优化错题集：
```
整理一下错题集
```

系统会自动：
- 检查并标记失效的路径
- 提醒长期未验证的记录
- 分析高频问题模块
- 提供优化建议

## 核心概念

### 自动记录

当以下情况发生时，AI 会自动记录问题：
- 用户指出错误并要求修正
- 用户反馈"之前不是这样的"或类似问题重现
- AI 发现代码存在潜在问题

### 智能召回

系统通过文件路径模式自动关联问题和代码：
- **路径匹配**：基于文件路径和通配符模式（如 `auth/*`）自动召回相关问题

### 问题评分

每个问题根据多个维度自动评分（0-100+），包括：
- 功能重要性、逻辑复杂度、影响范围
- 修复难度、出现次数、用户情绪、预防价值

### 验证机制

- **低分问题**（<30分）：自动验证，快速记录
- **高分问题**（≥30分）：需要人工确认和验证

## 技术架构

- **数据存储**：SQLite 本地数据库
- **技能系统**：基于 Claude Code Skills 机制
- **自动化**：集成到 AI 工作流程，无需额外操作

## 适用场景

✅ 适合个人长期开发的项目，特别是：
- 业务逻辑复杂的项目
- 需要长期维护的项目
- AI 经常参与编码的项目

❌ 不太适合：
- 多人协作项目（数据库不共享，每人独立）
- 一次性或短期项目
- 极其简单的项目

## 注意事项

- 数据库是项目级别的，不同项目互不干扰
- 定期整理可以保持错题集的健康度
- 代码重构后，整理功能会检测并标记失效的路径记录
