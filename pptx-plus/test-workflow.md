# PPT 分享生成工作流测试指南

本文档验证插件完整覆盖 spec 中的所有行为。对应 spec：`docs/superpowers/specs/2026-04-14-ppt-report-plugin-design.md`。

## 测试检查清单

### 输入格式

- [ ] **纯文本输入**：`请帮我做一个关于Q1工作分享的PPT`
- [ ] **文件路径输入**：`基于 D:/tmp/test-report.txt 生成分享PPT`
- [ ] **文件夹路径输入**：`基于 D:/docs/quarterly/ 文件夹生成分享PPT`
- [ ] **飞书文档链接输入**：`基于 https://xxx.feishu.cn/docx/xxx 生成分享PPT`
- [ ] **多来源混合输入**：文本 + 文件 + 飞书文档同时提供

### 工作流步骤

- [ ] **第 1 步**：正确解析输入，提取主题，读取工具指南
- [ ] **第 2 步**：主agent直接生成大纲并呈现
- [ ] **第 3 步**：用户确认大纲后立即保存（确认=保存，无独立步骤）
- [ ] **第 4 步**：询问用户选择丰富级别
- [ ] **第 5 步**：主agent直接丰富内容
- [ ] **第 6 步**：以 Markdown 格式预览内容，用户确认后立即保存（确认=保存，无独立步骤）
- [ ] **第 7 步**：主agent直接生成 PPT 保存到 output 目录
- [ ] **第 8 步**：报告完成

### 保存时机

- [ ] **大纲确认时**：原始材料已写入 `~/.pptx-plus/<话题>/input/`
- [ ] **大纲确认时**：`outline.md` 已写入 input 目录
- [ ] **大纲修改后**：`outline.md` 同步更新
- [ ] **内容确认时**：`content.md` 已写入 input 目录

### 丰富级别

- [ ] **L1 精简**：每节 1-2 句概述，仅 AI 常识
- [ ] **L2 标准**：每节 3-5 要点，AI + 用户材料
- [ ] **L3 丰富**：详细展开，AI + 用户材料 + 网络搜索
- [ ] **L4 定制**：行业数据、竞品对比，AI + 用户材料 + 深度搜索

### 内容确认

- [ ] **第 4 步**：询问丰富级别
- [ ] **第 5 步**：主agent直接丰富内容
- [ ] **第 6 步**：以 Markdown 格式预览内容
- [ ] **第 6 步**：内容来源标注正确（`[用户提供]`、`[AI知识]`、`[网络搜索]`）
- [ ] **内容修改后再次确认**：修改意见被正确应用

### 最终保存与输出

- [ ] **第 7 步**：PPT 保存到 `~/.pptx-plus/<话题>/output/`
- [ ] **第 7 步**：用户指定路径时优先使用指定路径
- [ ] **第 8 步**：报告完成，告知输出路径和 input 目录可复用

### PPT 质量

- [ ] 幻灯片数量 8-15 页
- [ ] 避免纯 bullet 幻灯片，布局多样化
- [ ] 包含视觉元素（图表、图标、数据卡片等）
- [ ] 开篇有执行摘要，结尾有行动项/建议页

## 测试场景

### 场景 A: 完整流程（纯文本）

1. 输入：`请帮我做一个关于Q1工作分享的PPT`
2. 验证：生成合理的大纲
3. 验证：大纲确认后可选择丰富级别（L1-L4）
4. 验证：内容丰富后生成 PPT
5. 验证：PPT 保存到 output 目录

### 场景 B: 带文件输入 + 内容修改

1. 输入：`基于 D:/tmp/test-report.txt 生成分享PPT`
2. 验证：正确读取文件内容
3. 验证：基于文件内容生成大纲
4. 验证：大纲确认后原始材料和 outline.md 写入 input 目录
5. 验证：内容预览后可修改
6. 验证：content.md 写入 input 目录

### 场景 C: 大纲修改 + 同步更新

1. 输入：`生成一个技术分享PPT`
2. 验证：生成大纲
3. 用户修改大纲（如增减章节）
4. 验证：`outline.md` 同步更新到 input 目录

### 场景 D: 多输入格式

1. 输入：`结合 D:/docs/report.txt 和 https://xxx.feishu.cn/docx/xxx 生成分享PPT`
2. 验证：文件内容和飞书文档内容均被读取
3. 验证：原始材料完整保存到 input 目录

## 手动测试步骤

1. 启动 Claude Code 并加载此插件
2. 执行 `cd D:/yeyi/AI/pptx-plus`
3. 按场景输入测试命令，观察各步骤输出
4. 检查 `~/.pptx-plus/<话题>/input/` 是否包含：原始材料、`outline.md`、`content.md`
5. 检查 `~/.pptx-plus/<话题>/output/` 是否生成 PPT 文件
