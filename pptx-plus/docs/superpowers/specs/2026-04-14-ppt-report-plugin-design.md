# PPT 分享生成插件设计

## 概述

开发一个 Claude Code 插件，帮助用户根据简略资料生成可用于团队分享的 PPT。AI 利用自身知识和网络搜索补充丰富资料，与用户探讨确认，最终输出专业 PPT。

## 核心流程

```
用户输入 → 大纲生成 → 用户确认(保存原始材料+outline) → 丰富级别询问 → 内容丰富 → 内容预览确认(保存content) → PPT 生成 → 发送
```

共 8 步，两次用户确认点（大纲 + 内容预览），用户始终掌控方向和深度。生成 PPT 前先预览内容。

## 输入格式

| 类型 | 处理方式 |
|------|---------|
| 自然语言文本 | 直接解析 |
| 文件路径 | 读取文件内容 |
| 文件夹路径 | 遍历所有文本文件，每个文件独立保存（不合并） |
| 飞书文档链接 | feishu-doc skill 读取 |
| 多来源混合 | 合并所有来源统一分析 |

## 丰富级别

| 级别 | 名称 | 说明 |
|------|------|------|
| L1 | 极简 | 仅大纲结构 |
| L2 | 基础 | 包含核心内容 |
| L3 | 标准 | 内容较完整 |
| L4 | 详细 | 深度扩展 |

> 详细数据来源说明（L1-L4 均基于用户材料）：L3/L4 级别需触发网络搜索补充数据。

│   8步流程，关键节点暂停   │

### 组件职责

- `ppt-report`（skill） — 工作流编排 + 主agent直接执行所有步骤，关键节点暂停等待用户确认
- `content-enricher`（工具指南） — 主agent在STEP1读取作为工具指南，供后续丰富内容使用
- PPT 工具（`tools/pptx/`） — 主agent在STEP1读取作为工具指南，STEP7直接使用生成PPT
- metabot — 飞书文件发送（复用）
- feishu-doc — 飞书文档读取（复用）

## 触发方式

`ppt-report` skill 使用宽泛触发描述，截获所有 PPT 请求。当用户提到"PPT"、"演示文稿"、"deck"、"slides"、"presentation"或任何.pptx文件时，自动激活该skill。所有步骤由主agent直接执行。

## 输出

- **输入保存**：`~/.pptx-plus/<话题>/input/` 下保存：本地文件/文件夹（每个独立文件，使用原文件名）、`links.txt`（所有链接列表）、`outline.md`（大纲确认时写入，后续修改同步）、`content.md`（内容预览确认后立即写入）
- **工作目录 + 产物**：`~/.pptx-plus/<话题>/output/`，运行时统一使用此目录，PPT 产物和中间文件均存放在此。**仅当用户明确指定输出路径时**，PPT 文件发送到指定位置，其他中间产物仍留在工作目录。
- **话题 slug**：中文话题翻译为英文语义，小写，空格变 `-`，特殊字符变 `_`（如"AI发展趋势"→`ai-development-trends`）
- **飞书**：对话中直接发送 PPT 文件

## 用户体验

1. 用户输入主题（可附文件/文件夹/飞书文档链接）
2. AI 解析输入，形成输入摘要，生成大纲呈现给用户
3. 用户确认 / 修改 / 补充大纲，**确认后立即写入原始材料和 outline.md 到 input 目录**
4. AI 询问丰富级别（L1-L4）
5. AI 按级别丰富内容（L3/L4 需网络搜索）
6. AI 呈现丰富后内容供用户预览确认 / 修改，**确认后立即写入 content.md 到 input 目录**
7. AI 生成 PPT 保存到 output 目录
8. AI 报告完成，通过飞书发送 PPT 文件，提示可换路径

> **注：** 确认和保存为同一原子操作，确认后立即执行保存，无独立保存步骤。
